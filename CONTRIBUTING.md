# Contributing

Thank you for your interest in contributing to GitHub Dotfiles!

## How to Contribute

1. Fork the repository on GitHub.
2. Create a new branch for your feature or bug fix.
3. Make your changes, ensuring they follow the project's conventions.
4. Test your changes thoroughly.
5. Submit a pull request with a clear description of the changes.

## Guidelines

- Follow the existing code style and conventions.
- Write clear, concise commit messages following the conventional commit format.
- Ensure all pre-commit checks pass.
- Update documentation as needed.

## Reporting Issues

If you find a bug or have a feature request, please open an issue on GitHub with detailed information.

## Code of Conduct

Please be respectful and constructive in all interactions.
